# 嵌入式行业分工地图

> 承接 [技术全景](/overview/0_layered_arch)：四层链讲的是「系统怎么连」，这一篇讲的是「人怎么分工」——嵌入式行业里都有哪些工种、你的目标岗位在哪。

嵌入式**没有唯一的分法**。行业里主要有两条切分轴，而**拆分的粗细取决于公司规模**。

## 一、按职能层次分（主流）

### 硬件侧（EE）

| 岗位 | 干什么 | 门槛 |
|---|---|---|
| 系统 / 架构 | 整机方案、选型、框图 | 高（资深） |
| 数字硬件 | 处理器周边、总线、FPGA | 常规，最低 |
| **模拟硬件** | 前端采样、信号调理 | 高 · 专精 |
| **电源 / 功率电子** | 电源设计、大电流、散热 | 高 · 专精 |
| **高速信号 / SI** | DDR/以太网布线、阻抗、仿真 | 高 · 专精 |
| 射频 RF | 无线、天线、微波 | 很高 · 专精 |
| PCB Layout | 布局布线 | 中（大厂独立工种） |
| 硬件测试 / 可靠性 | DVT、EMC、环境/寿命 | 中 |

::: tip 「硬件难点方向」这种说法
实际工作中常听到把硬件按**高门槛专精方向**归纳成「大功率 / 高速信号 / 模拟电源」三块——它对应的正是上表里的**电源、SI、模拟**三个专精岗。这不是 MECE 的完整分类（常规数字/接口没算在内，且彼此交叠），而是「硬件里最难、最需要专家的地方」的实用概括。
:::

### 软件侧（嵌入式软件 / 固件）

嵌入式软件不是铁板一块，最有用的分法是按**离硬件的远近**排一条光谱——这条线也决定了 Java 背景的人该站哪：

```
离硬件最近 ◄──────────────────────────────────────► 离硬件最远(最像后端)

裸机固件  →   RTOS   →  BSP/驱动/内核  →  Linux系统  →  Linux应用层
寄存器编程    实时任务    U-Boot/设备树/     内核裁剪/      网络/协议/
无OS         FreeRTOS   驱动移植          Buildroot/Yocto  数据处理/上云
└──── MCU 组 ────┘      └────────── Linux 组 ──────────────────┘
                        (BSP/驱动方向)        (应用方向,最佳落点)
```

| 岗位 | 干什么 |
|---|---|
| 固件 / MCU 工程师 | 裸机、RTOS、外设驱动、实时控制 |
| BSP / 驱动工程师 | bootloader、内核移植、设备树、驱动 |
| 嵌入式 Linux 系统 | 内核裁剪、Yocto/Buildroot、根文件系统 |
| 嵌入式应用 | 应用逻辑、网络、协议、中间件、上云 |
| 上位机 / 工具软件 | 配置调试工具、组态、HMI |

其余支撑岗：软/硬/系统测试、认证（EMC / 安规 / **计量认证**）、结构散热、项目管理、FAE（现场应用工程师）。

## 二、拆多细，取决于公司规模

这是「行业怎么分」没有标准答案的根本原因：

| 公司规模 | 怎么分 | 例子 |
|---|---|---|
| 大厂 | 拆到极细，每个方向独立组甚至独立部门 | 华为 / 海康 / 大疆：硬件分数字/模拟/电源/射频/SI/Layout；软件分 BSP/驱动/系统/应用/测试 |
| **中型设备公司** | **按大块分组** | **硬件 / MCU / Linux / 上位机（本项目背景所在）** |
| 小公司 / 创业 | 一个「嵌入式工程师」软硬通吃 | 从画板到写固件到调 Linux 全包 |

::: info 中型公司的四分法是标准结构
「硬件 / MCU / Linux / 上位机」就是中型工业设备公司的通行分组，和行业主流完全一致。大厂只是把其中的「Linux 组」再拆成 BSP/驱动 + 系统 + 应用而已。各组与四层链的对应关系见 [技术全景](/overview/0_layered_arch)。
:::

## 三、另一条轴：按应用领域分（行业垂直）

同样是「嵌入式」，不同行业味道差很远，深度要求也不同：

| 领域 | 核心诉求 |
|---|---|
| 消费电子（手机/可穿戴/家电） | 成本、功耗、量产 |
| **工业控制 / 能源计量** | 精度、可靠性、**计量认证**、电气安全 |
| 汽车电子 | 车规、功能安全 ISO 26262、AUTOSAR |
| 通信设备（路由器/交换机） | 高速信号、协议栈 |
| 物联网 / 表计 | 低功耗、联网、云平台 |
| 医疗 / 航空航天军工 | 超高可靠 |

## 四、回到你：目标岗位在行业里叫什么

- 你的目标岗位，行业标准叫法是 **「嵌入式 Linux 工程师（应用 / 网关方向）」** 或 **「IoT / 工业互联网工程师」**——横跨 Linux 组和云平台，是那个把「设备到云端」整条链打通的桥梁角色。
- 沿软件光谱看，你从**最右端的 Linux 应用层**切入（最像后端），再逐步**向左延伸**到驱动 / BSP。本站主线（basics → C → 系统编程 → 网络 → 嵌入式 → 驱动 → 内核）就是带你从光谱右端一路向左走。
- 硬件那三个专精方向（电源 / SI / 模拟）是行业里门槛最高的硬件岗，你只需**懂到能对话**，不转——这也是本站把 hardware 定位成「延伸」的原因。
